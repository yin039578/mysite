## 作業流程


## 流程
### 發送信件
1. 後台>銷售管理>課程通知(新增)>課程通知功能>新增通知按鈕
2. 輸入信件內容(主旨、內容)、選擇課程，課程依照id或名稱搜尋
3. 按下發送後即時發信給該課程所有已成功報名的人

### 查詢發送紀錄
1. 後台>銷售管理>課程通知(新增)>課程通知功能
2. 按照輸入條件篩選課程，條件包含廠商(下拉)、課程id、廠商課程id、課程名稱、發送時間區間、信件主旨、課程類型(checkbox多選)篩選
3. 寄送紀錄列表提供:課程id、廠商id、課程名稱、課程類型、信件主旨、發送人數、發送成功數、發送時間、發送人、功能(檢視)，列表有分頁功能
4. 點擊列表功能(檢視)，呈現信件主旨、內文、會員列表，會員提供篩選功能
 
## 開發範疇
### 後端
#### 建立新資料表：課程發送紀錄批次
```
CREATE TABLE `nabi_mail_batch` (
`id` int unsigned NOT NULL AUTO_INCREMENT COMMENT '發信流水號',
`provider_id` int unsigned NOT NULL COMMENT '來源',
`course_id` int unsigned NOT NULL COMMENT '流水號',
`provider_uid` varchar(100) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci NOT NULL COMMENT '第三方課程自家系統的唯一編號',
`mail_title` varchar(500) COLLATE utf8mb4_unicode_ci NOT NULL COMMENT '標題',
`send_count` tinyint NOT NULL DEFAULT 0 COMMENT '發送數量',
`send_success_count` tinyint NOT NULL DEFAULT 0 COMMENT '發送數量',
`create_date` datetime NOT NULL COMMENT '發送日期',
`create_user` varchar(50) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci NOT NULL COMMENT '發送者',
PRIMARY KEY (`id`)
);
```
#### 新增 發送課程通知API
1. 檢查參數與內容
2. 建立發信時間，建立發送批次
3. 以發信時間為準抓出課程報名成功的會員列表
4. 逐一發信寫入發信紀錄
5. 發完之後更新批次的發送數量與成功數量。因為數量不大，先不考慮分散式發送，一條龍執行，也比較好統計數量跟錯誤處理。
